STM8S/A Standard Peripherals Library: Peripheral’s examples

 

Copyright © 2014 STMicroelectronics

 

 

The STM8S/A Standard Peripherals library provides a rich set of examples covering the main features of each peripheral. 
Almost exmaples run on all supported STM8S/A devices. However some of them are device specific due to peripheral availability constraint.
All the examples are independent from the software toolchain. These examples run on STMicroelectronics STM8/128-EVAL evaluation board and can be easily tailored to any other supported device and development board. 

Only source files are provided for each example and user can tailor the provided project template to run the selected example with his preferred toolchain.

What do I need to start successfully with Examples?

Evaluation Board: STMicroelectronics STM8\128-EVAL evaluation board or any other compatible hardware. To use your own specific hardware, you need simply to adapt the examples hardware configuration to your platform.

Development Toolchain: 

ST Visual Develop STVD (v4.3.0 or later) used with

  Cosmic compiler (v4.3.6 or later)

  Raisonance STM8/ST7 C compiler (v2.38 or later)

 Raisonance Integrated Development Environment RIDE7 (v7.36.)used with STM8/ST7 C compiler (v2.38 or later)  

IAR Embedded Workbench for STM8 (EWSTM8)  (v1.3.0) used with IAR C/C++ compiler for STM8

Debug Instrument: 

STVD: Swim ST-Link, Swim STice or Swim Rlink. Swim ST-Link is selected by default in the template project, to modify this configuration select “Debug instrument” menu then in  “Target” window select the debug instrument you want to use.

RIDE7: Swim Rlink

EWSMT8:  Swim ST-Link

How to proceed?

Copy the files main.c, main.h (if exists), stm8s_it.c, stm8s_it.h, stm8s_conf.h and any other source files from the specified example to the template project directory (Project\Template). This project folder contains already all toolchain files needed for creating projects for STM8S/A microcontrollers.

Open the project workspace according to the toolchain you will use:

 Project\Template\STVD\Cosmic\project.stw

 Project\Template\STVD\Raisonance\project.stw

 Project\Template\RIDE\project.rprj

 Project\Template\EWSTM8\project.eww

Set active the project related to the STM8S or STM8A used device

stm8s208 :  to configure project for the STM8S High-density devices with CAN

stm8s207 :  to configure project for the STM8S High-density devices without CAN

stm8s007 :  to configure project for the STM8S Value Line High density devices without CAN

stm8s105 :  to configure project for the STM8S Medium-density devices

stm8s005 :  to configure project for the STM8S Value Line Medium-density devices

stm8s103 :  to configure project for the STM8S Low-density devices

stm8s003 :  to configure project for the STM8S Value Line Low-density devices

stm8s903 :  to configure project for the STM8S Low-density devices

stm8af52ax :  to configure project for the STM8A High-density devices with CAN

stm8af62ax :  to configure project for the STM8A High-density devices without CAN

stm8af626x :  to configure project for the STM8A Medium-density devices

Add the needed Library sources files (including peripheral’s drivers) according to the “How to use it” section of the readme file

Finally compile, load and run the example
